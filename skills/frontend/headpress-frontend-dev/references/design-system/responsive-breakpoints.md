# 裝置與響應式斷點 (Responsive Breakpoints)

前端設計系統的裝置相容性必須涵蓋以下四種主要裝置。請在開發 UI 元件與排版時，確實遵守這些裝置的響應式斷點 (Breakpoints) 與體驗規劃。

## 支援裝置清單

1. **大螢幕 (Large Screens / Desktop)**
   - 針對一般桌上型電腦、筆電與寬螢幕顯示器。
   - 應確保大螢幕下的最大寬度 (max-width) 限制、版面配置最佳化、以及 hover/focus 等滑鼠互動行為正常運作。

2. **小螢幕 (Small Screens / Laptops)**
   - 針對較小尺寸的筆電或視窗縮小的情境。
   - 應確保內容不溢出，複雜元件與多欄佈局在空間不足時能平滑且優雅地適應或縮放。

3. **平板 (Tablets)**
   - 針對 iPad 等一般平板裝置。
   - 需同時考量橫向 (Landscape) 與直向 (Portrait) 顯示，確保觸控熱區 (Touch targets) 大小適中（建議至少 44x44px）。佈局可能從多欄切換為較緊湊的卡片式或網格系統。

4. **手機 (Mobile Phones)**
   - 針對各類智慧型手機。
   - 採用單欄佈局為主，導覽列需改為抽屜式 (Drawer)、漢堡選單 (Hamburger menu) 或折疊面板 (Accordion)。確保文字大小易讀，完全支援觸控操作與單手使用習慣。

## 實作原則

- **流暢縮放與過渡**: 在各個斷點之間切換時，元件不應重疊、文字不應被切斷。請盡量使用相對單位與 flex/grid 彈性排版，而非依賴定死的高度與寬度。
- **測試與驗證**: 每次提交 UI 變更前，開發者或 Agent 都必須確認大螢幕、小螢幕、平板與手機這四種裝置情境的渲染狀態正常。
