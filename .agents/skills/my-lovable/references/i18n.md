# I18n

## 目錄規劃

```text
src/i18n/
  config.ts
  keys.ts
  locales/
    en.json
    zh-TW.json
    ja.json
```

內容型 CMS 網站中，CMS content 留在 CMS，locale files 主要存 UI chrome。

## Key 規則

- 使用穩定語意 key：`nav.home`、`product.card.priceLabel`。
- 不要用英文原文當 key。
- 既有專案已有 locale files 時，不要 hardcode 新可見文字。
- 要有 fallback language 行為。
- 日期、數字、貨幣透過 locale-aware utilities 格式化。
- SSR context 不要在 initial render 讀 `localStorage`。

## Locale JSON Example

```json
{
  "nav.home": "Home",
  "nav.news": "News",
  "common.loading": "Loading",
  "common.error": "Something went wrong"
}
```

## Prompt Guardrail

```text
This project uses i18n.
Add or update locale keys in src/i18n/locales.
Do not hardcode new visible UI text in components.
If language preference uses browser storage, guard it for SSR.
```
